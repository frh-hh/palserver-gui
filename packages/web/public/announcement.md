id: 2026-07-coop-worldoptions
title: Co-op (4-player) saves on a dedicated server: settings not applying / admin password mismatch
lang: en
---

If you migrated a **co-op (4-player) save** to a dedicated server, you may have hit these issues:

- World settings changes **don't take effect**
- You've set an admin password, but the player list shows "**admin password may not match**"

## The cause

Co-op saves leave behind a `WorldOptions.sav` file. When the dedicated server finds it, it **applies that file and ignores the GUI-managed config** (including the admin password) — so everything you change in the GUI gets overridden.

## The fix (current version)

1. Open the "**Saves & Backups**" tab and click "**Browse files**" on that world to open its save folder
2. Delete (or rename) the **`WorldOptions.sav`** inside
3. **Restart the server** — world settings and the admin password will now apply properly

## Next version

Will **detect this automatically and offer a one-click fix**: handled automatically when importing co-op saves, and existing worlds will show a warning with a repair button on the Saves tab.

Sorry for the trouble, and thanks for the reports!
